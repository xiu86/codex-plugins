# 安全策略

## 支持版本

安全修复优先面向本插件的最新发布版本。

## 漏洞报告

请勿通过公开 issue 报告敏感漏洞。请通过 `.codex-plugin/plugin.json` 中列出的维护者联系方式，或代码托管平台的私密安全报告功能提交。

报告中建议包含：

- 受影响的 Skill 或文件。
- 复现步骤。
- 影响范围。
- 已知的缓解建议。

## 安全范围

本插件用于指导 Codex 执行接口黑盒测试。安全相关风险包括：

- 提示词引导用户在生产环境执行不安全测试。
- 指令要求在日志或报告中暴露 secret。
- 未经确认就允许破坏性数据库变更。
- 报告在缺少证据时将安全测试标记为通过。
- 测试 payload 不适合共享环境或非隔离环境。

## 安全使用要求

- 破坏性或侵入式 payload 应使用隔离测试环境。
- 报告中必须脱敏 token、cookie 和凭证。
- 未经授权不得对系统执行安全测试。
- 只在授权环境中验证数据库写入。
- 缺少证据时应标记为阻塞或证据不足，而不是通过。
